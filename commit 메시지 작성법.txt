# 각 부분에 들어가야할 내용과 형식

* Type
- feat : 새로운 기능을 추가할 때 
- fix : 버그를 고칠 때
- docs : 문서를 수정할 때
- style : 코드 스타일 또는 포멧 등을 수정할 때
- refactor : 프로덕션 코드 리팩터링
- test : 테스트 추가, 테스트 리팩터링(프로덕션 코드 변경 없음)
- chore : 빌드 테스크 업데이트, 패키지 매니저 설정할 경우 (프로덕션 코드 변경 없음)

* title
- (영어) 첫 단어는 동사 원형, 대문자로 시작, 명령어 사용
ex) docs(changelog): remove unnecessary revert item
- (한글) 명사형 문장으로 작성
ex) Feat : 관심지역 알림 on/off 기능 추가


* Body 
본문. 상세 내용을 작성하는 곳. 한줄당 72글자 넘어가지 않게
Type : title만으로 모두 설명이 되었다면 안써도 됨.
 

* Footer

이슈 트래커 ID가 없다면 안써도 됨.
